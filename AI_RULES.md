# AI_RULES.md

## Code Modification Rules
1. **Context Awareness**: Always read `APP_STRUCTURE_REPORT.md` and `SERVER_MAP.md` before applying changes to understand module boundaries.
2. **Isolation**: Keep Brand management, AI generation, and WordPress integration in separate modules/files. Do not merge them into a single massive `server.ts` or `App.tsx`.
3. **Draft First**: Never change the default WP post status from `draft` to `publish` unless explicitly requested by the user per-request.
4. **Validation**: Every AI-generated piece must be validated for length and "buy-intent" markers on the client-side before submission to Backend.
5. **No Hallucination**: If the brand data is missing "real deals" or "coupon page" status, the AI must NOT invent specific coupon codes. It should focus on general "how to save" guides.
6. **Module Integrity**: When adding features, prefer adding new files in `src/components/` or backend routes rather than bloating existing ones.
9. **Multi-Point CTA**: Every generated article MUST include 3-5 CTAs. Shopping-related CTAs must prioritize the `affiliate_url` with correct `rel="nofollow sponsored"` attributes.
10. **Explicit Reasons**: If a file is modified, the commit message or summary must explain *why* and which module it belongs to.
11. **HTML Safety**: All content generated by AI must be sanitized before being displayed or sent to WordPress.
